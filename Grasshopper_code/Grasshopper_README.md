# Grasshopper Code

This folder contains examples of parametric design tools built in Grasshopper. Each file comes with a description or tutorial explaining its inputs, outputs, and intended use.

## Purpose

The goal of this folder is simple: to help students help each other. Parametric design has a steep learning curve, and a lot of useful work gets lost after a project ends. By sharing scripts and workflows here, we build something that grows with us — a library shaped by real problems and real solutions from within our own community.

## How to Use

1. Browse the folders and find a tool relevant to your work
2. Download the `.gh` or `.pdf` file
3. Follow the included instructions or tutorial
4. Adapt it freely to your own project

## How to Contribute

If you have a script you want to share, there are two ways to do it:

- **Fork** this repository, add your file inside a folder along with a short description or tutorial, and submit a **Pull Request**
- **Email us** at uthopen@gmail.com and we will add it for you

If you have been working on someone else's script and found improvements, we encourage you to share those too. No contribution is too small — a better input method, a cleaner workflow, or a fix for a known issue all make the library more useful for everyone.

## What We Are Building

This is meant to be a living library, not an archive. Every script added, every improvement shared, and every problem solved here makes the next project a little easier for someone else. The more it grows, the more useful it becomes — for all of us.
